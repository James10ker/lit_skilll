# Step 6: 综述正文撰写

目标：写出可交付的文献综述正文，而不是研究计划、开题报告、检索方案或论文摘要拼接。

## 交付边界

- 标题应是综述题目，例如“检索增强生成系统中的幻觉抑制研究综述”，不要加“研究计划”。
- 正文应包含连续论述段落，不能只输出研究问题、检索关键词、纳入排除标准和章节安排。
- 检索策略、纳入排除标准只在系统综述/范围综述或用户明确要求方法透明度时简写为一节；普通文献综述应把篇幅用于研究现状、比较综合和未来方向。
- 如果缺少真实文献来源，不要编造引用；可以写“待检索补齐”版综述框架，但必须明确这不是最终可引用正文。

## 推荐结构

普通综述可使用：

1. 引言：研究背景、问题重要性、综述范围和本文综述视角。
2. 概念与理论基础：定义、边界、核心概念关系。
3. 文献检索与筛选方法：适用于系统综述/范围综述；普通课程综述可简化。
4. 主题综述：按问题、方法、应用或时间线组织。
5. 比较与综合：方法优劣、证据强弱、适用条件、争议。
6. 研究不足与未来方向：从证据中推导，不空泛罗列。
7. 结论：回应研究问题，概括领域现状。

参考综述复现式生成应使用：

1. 引言：说明研究领域和复现参考综述的目的。
2. 数据来源与方法：说明参考综述来源、检索范围、筛选方式，以及本次可访问来源的差异。
3. RQ1：年度发文数量。
4. RQ2：主要发表来源、国家/地区和机构。
5. RQ3：国家和机构合作关系。
6. RQ4：主要研究主题。
7. RQ5：主题研究兴趣变化。
8. 结论。

若用户明确要求复现实验、效果评估或“与参考综述对比”，可在结论前后加入效果对比节，比较 RQ 覆盖、图表类型、主题分析和引用标注。若只是生成普通单篇综述，不要强制加入对比章节。

## 写作要求

- 最终成稿全部使用英语，包括标题、摘要、关键词、章节标题、正文、表格文字、图题、图注及所有图形内文字。
- 段落主题句明确，段内按“观点 -> 证据 -> 综合判断”推进。
- 方法部分报告全文、摘要和仅元数据记录的覆盖率；结论强度必须与证据层级匹配，不能把摘要级分析描述为对全部论文的全文综合。
- 引用紧贴具体观点；不要把一串引用堆在段末。
- 对不一致结论要解释可能原因：数据、样本、评价指标、任务设定、时间背景。
- 普通综述可主动加入必要图表：分类框架、技术路线、时间线、代表性文献对比表、数据集/指标表、研究空白矩阵。图表是按需增强，不是每篇综述的硬性数量指标。
- 参考综述复现式生成时，图表只做“图表合成”：用本次文献统计数据生成与参考综述同类型、同功能的新图，例如趋势图、关系图、饼状图、流程图或思维导图。
- 计划书中的流程图/思维导图、趋势图、关系图、饼状图是必须支持的图表类型；只有用户要求、RQ 需要且数据足够时才生成，不要为了凑齐类型而编造图表。
- 参考综述复现式生成时，不做参考图表内容分析，不默认抽取或复制参考论文原图。
- 最近五年窗口必须截止于最新完整年份；当前年或其他不完整年份只能单独披露，不能混入完整窗口比较。
- 主题分析先建立代码到唯一英文展示标签的映射，并在所有统计和图表中复用该映射。
- 合作关系结论与网络图必须由真实共著、机构或国家字段支撑；字段不足时删除该图和相应定量结论，改为局限说明。
- 需要表格时优先呈现 RQ 所需统计表：来源排名、国家/地区排名、机构排名、主题分布、筛选记录。
- 需要图示时可用 Mermaid 或 Python 图表；图题必须说明数据来源和对应 RQ。
- 每张图表前后都要有正文解释，不能孤立堆放。正文中必须引用图表编号，例如“见表 1”“如图 2 所示”。

## 落盘

默认保存为完整 LaTeX 工程：

- `outputs/{主题标识}/review.tex`
- `outputs/{主题标识}/references.bib`
- `outputs/{主题标识}/figures/`
- `outputs/{主题标识}/data/evidence_ledger.json`
- `outputs/{主题标识}/reports/`

读取 `prompts/latex_delivery.md` 并执行静态质量门禁。默认不把 Markdown/Word 作为主交付物。只有用户明确要求 Word 时，才额外生成 `.docx`；不得用 Word 交付替代 LaTeX 源稿。

若用户明确要求 Word，则：

- Markdown 只作为内部中间稿使用。
- 必须运行 `tools/md_to_docx.py` 生成 `.docx`。
- 交付说明中只突出 `.docx` 路径；不要把 `.md` 作为主交付物。
- 若文中含 Mermaid 图，优先先用 `tools/mermaid_render.py` 渲染；若渲染环境不可用，说明 Word 中 Mermaid 可能以代码块保留，或改用已生成的 PNG/表格替代。

不要在文件名中加入“研究计划”，除非用户明确要求研究计划。

不要覆盖旧稿，除非用户明确要求。
