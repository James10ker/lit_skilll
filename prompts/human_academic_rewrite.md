# Step 8.5: 人工学术风格重写

目标：把已完成证据链自检的综述草稿改写成更接近人工学术写作的正文。该步骤提升可读性、论证自然度和学术可信度，不用于隐藏 AI 使用，也不得包装不可核验内容。

## 前置条件

重写前必须先完成事实修正：

- 正文纳入文献数、`screening_table.csv` 行数、图表数据表总数一致。
- 每张图的来源与正文表述一致；估算值、二级综述汇总、人工编码必须如实说明。
- 关系图、趋势图、饼图等图表有对应数据文件；不能把脚本手写矩阵说成自动统计。
- 不存在明显编造的作者、年份、DOI、会议、期刊或实验结果。

若前置条件不满足，先修证据链，不要直接润色。

## 重写原则

- 重写后的题名、摘要、关键词、正文、表格及所有图形文字必须全部为英语。
- 保留真实事实、引用、图表编号和数据限制。
- 不新增未经核验的文献、数据、DOI、作者、年份或结论。
- 不把“待核验”内容改写成确定事实。
- 不把估算数据、二级综述汇总或人工编码包装成精确统计结果。
- 不写“为了显得像人工”的自我说明。

## 去除 AI 模板感

重点删除或改写以下痕迹：

- 机械句式：`该图用于回答 RQx`、`本文系统性地...`、`核心挑战包括...`、`最终输出...路径如下`。
- 清单填空：每节都按同一种 “定义 -> 分类 -> 代表论文 -> 小结” 模板展开。
- 过度粗体：除标题、小标题和必要术语外，不要大面积加粗。
- 过多项目符号：能写成连续论述的列表应改成段落。
- 空泛评价：`显著增长`、`高度活跃`、`重要意义` 等应补证据或改成审慎表述。
- 重复限制声明：数据限制集中放在方法或结论中的一段，避免每节反复打断正文。
- prompt 痕迹：不要出现“本次测试”“必须生成四类图表”“输出路径”“对应 RQ”等执行指令口吻。

## 人工学术写法

正文应呈现一条自然主线，而不是 RQ 的机械答案堆叠。

可采用：

- 引言中提出领域演化主线和综述必要性。
- 方法部分简洁交代检索、筛选、编码和数据限制。
- 分析部分围绕时间演化、主题分化、方法转向、合作格局和争议展开。
- RQ 可以保留为结构线索，但段落要自然回应，不要写成问答模板。
- 图表自然嵌入论述，例如“图 2 显示...”或“从图 3 的合作网络可以看出...”。
- 对不确定结果使用审慎表达，例如“现有公开来源显示”“在本次样本范围内”“该趋势仍需数据库级检索验证”。

## 图表文字改写

图注和正文说明应学术化：

错误示例：

```text
图 2 用于回答 RQ1。
```

推荐：

```text
图 2 汇总了公开文献计量研究中报告的 AIEd 年度发文趋势。由于 2025-2026 年包含估算或不完整年份数据，该图更适合用于观察增长方向，而不宜解释为精确年度统计。
```

错误示例：

```text
图 3 的边权重反映合作频率。
```

推荐：

```text
图 3 基于 `relationship_edges.csv` 展示国家/地区层面的合作关系。由于部分边权来自既有文献计量综述的汇总结果，而非逐篇共著记录的重新计算，该网络应被理解为合作格局的近似呈现。
```

## 输出要求

- 直接生成重写后的正文或文件。
- 不输出完整自检清单。
- 交付说明只简述：事实修正、风格重写、残余限制。
- 若无法完成事实修正，明确说明阻塞点，并保留为“待核验草稿”。
