# 图表合成

目标：根据本次检索和筛选得到的文献数据，按需生成与研究问题或参考综述同类型、同功能的图表。

本步骤不做像素级临摹：不复制参考论文原图或节点坐标，也不默认做“原图 + 重绘图”。但当用户要求“按论文形式”时，必须核对参考图的分析语义与视觉编码（节点代表什么、边代表什么、节点面积/线宽/颜色分别编码什么），再用本次数据生成同形式的新图。

## 图表合成的含义

图表合成指：

```text
参考综述有什么类型的图
-> 本次系统用自己检索/筛选得到的数据
-> 生成同类型、同用途的新图
```

示例：

- 参考综述图 3 是年度发文数量趋势图，则本系统统计本次文献集合中每年的文章数量，并生成新的趋势图。
- 参考综述图 7 是合作关系图，则本系统根据本次文献集合中的国家/机构共现关系生成新的关系图。
- 参考综述图 11 是饼状图，则本系统根据本次统计维度生成新的饼状图。

## 计划书与参考复现指定图表

需要支持：

1. 流程图：文献检索、去重、筛选、纳入和分析步骤。
2. 年度发文量柱状图：对应参考综述 Figure 3，深红柱、棕金趋势线、白底衬线字体。
3. 合作连接图：对应参考综述 Figure 7/8，仅表达真实的作者共著、机构或国家合作关系；使用力导向 node-link 形式，节点面积表示子集内发文量，线宽表示共同署名次数，颜色表示检测到的合作社区。
4. 双时间窗 topic 词云：用户指定截止年，比较最近五年与此前五年；两个窗口必须相邻且不重叠。默认使用更大的白色或浅色宽幅双面板、清晰的年份与记录数、克制配色和充足留白；在不重叠的前提下提高标签容量，避免把词语挤在画布中心。
5. topic-year 演化图或主题分布图：用于回答研究兴趣如何变化；参考综述 Figure 11 实际为年度主题比例图，不应误画成无时间维度的饼图。

参考综述复现任务默认生成流程图、年度图和双时间窗 topic 词云；合作字段充分时增加连接图。普通综述是否生成取决于用户要求、RQ 和数据字段。需要主题演化时增加第五类。

## 固定工具

- 流程图：`tools/render_review_figure1.py`
- 年度柱状图与双五年词云：`tools/render_temporal_topic_figures.py`
- 真实合作连接图：`tools/render_collaboration_networks.py`
- 异构文献计量图：`tools/render_bibliometric_network.py` 仅可用于明确命名的 bibliographic evidence map；不得用于复现 Figure 7/8 或回答 scientific collaboration

所有工具都必须输出 report，且 `validation.passed` 为 true。SVG 用于无损编辑；LaTeX 正文默认引用同名 PNG。

## 图表数据来源

图表只能来自本次文献集合的统计结果或主题分析结果，例如：

- 年份 -> 文章数量
- 来源 -> 文章数量
- 国家/地区 -> 文章数量
- 机构 -> 文章数量
- 国家/机构共现 -> 合作关系
- 主题 -> 文献数量
- 主题 + 年份 -> 主题热度变化

双五年 topic 图的 topic 必须来自作者关键词、数据库 concepts、人工编码或模型辅助编码台账。若从摘要/全文归纳，方法中必须标注编码方式并保留逐篇映射；不得把推断 topic 伪装成作者关键词。

生成图表前执行以下数据门禁：

- 所有可见文字必须为英语，包括标题、图例、坐标轴、节点、注释和分面标签。
- 将 topic code、缩写和展示名称归一化为唯一英文标签；同一概念只统计一次，主主题与次主题采用事先声明且一致的计数口径。
- 双五年窗口截止于最新完整年份。例如 2025 是不完整年份且 2024 为最新完整年份时，窗口应为 2015--2019 与 2020--2024；不得写成 2016--2020 与 2021--2025。
- 合作图必须由真实的 co-authorship、affiliation 或 country 字段构建。若这些字段缺失或覆盖不足，不生成合作图；不得用 topic、journal 和 year 的混合共现替代合作关系。
- 连接图 report 必须记录字段覆盖率、参与合作的多实体论文数、原始/可见节点数、边数、加权边数、筛选规则、节点/标签重叠和越界检查。若仅扩充子集有 affiliation 字段，图内徽标、图注和正文都必须披露 `n_with_field / n_included`，不得暗示全语料覆盖。
- 连接图不得用接近不可见的小点表示低频实体。report 必须声明最小节点面积、按最终 PNG DPI 换算的最小节点直径和最小标签字号，并把三项作为硬门禁；图进入论文前还必须在论文实际栏宽下目检可读。
- 机构/国家图默认按发文量、加权度和度数选取 top 30 左右的节点，并从诱导子图中去除孤立候选；筛选不允许新增边。作者图使用真实共著边，禁止把 author--topic 或 author--journal 边混入。
- 双五年词云的字体大小按各自窗口内频次缩放；除非报告明确使用同一分母，不得把两个窗口的绝对字体大小解释成跨窗可比。所有规范化 topic 标签应尽量完整落图，跳过项必须写入 report。
- 流程图必须用真实字体度量而非仅按字符数估算换行；report 对每个框记录可用宽高、最大实测行宽和文本总高，任何文字越框都判失败。统一使用四类克制语义色、10 px 圆角和 1.5 px 描边；主路径使用 2 px 实线，注释/排除分支使用 1 px 虚线；不添加装饰图标或图标占位列。

如果某张图所需字段缺失，必须说明“数据不足，无法合成该图”，不要编造数据。若某类图与当前主题或 RQ 无关，也应说明并跳过。

## 图表输出

图表应服务于 RQ。以下是可选对应关系，不要求每篇综述全部生成：

- RQ1：趋势图，展示年度发文数量。
- RQ2：来源、国家/地区、机构的排名图或表。
- RQ3：国家或机构合作关系图。
- RQ4：主题分布图或表。
- RQ5：主题热度变化图。

普通任意主题综述中，只生成能增强论证的图表。参考综述复现或用户明确要求时，再尽量覆盖参考综述指定的图表类型。

## 正文写法

每张图表必须包含：

- 图表编号
- 图表标题
- 数据来源说明
- 对应 RQ
- 一段解释文字

示例：

```text
图 3. 2000-2019 年 AIEd 文献年度发文数量趋势。
数据来源：本次从 WoS、Scopus、ERIC、ICAIED 与 IJAIED 检索并筛选得到的文献集合。
该图用于回答 RQ1。
```

## 禁止项

- 不默认抽取参考论文原图。
- 不默认插入论文原图。
- 不分析参考图表的视觉细节。
- 不把没有统计依据的数据画成图。
- 不为了满足固定数量而强制插入无关图表。
- 不声称复现了无法访问数据库中的精确结果。
- 不在英文论文中输出中文或中英双语图表标签。
- 不把 topic code 与其展示标签同时画入词云或统计为两个主题。
